How the Review Process Works

Every piece of content goes through an AI-assisted quality check before publishing.

1

Fork & Write

Fork the repo, create a Markdown file in _patterns/, _posts/, or _tutorials/.

2

Open a PR

Submit a pull request. The GitHub Action triggers automatically within seconds.

3

Claude Reviews

Claude AI checks technical accuracy, code quality, clarity, and structure, then posts detailed feedback on the PR.

4

Human Approval

A maintainer reviews Claude's feedback and your content, then merges to publish on GitHub Pages.
